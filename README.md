# Graph Visualizer - Адаптивная визуализация графов

Система интеллектуальной визуализации больших сетевых графов с автоматическим выбором алгоритма раскладки на основе анализа структурных метрик.

## Возможности

- **Адаптивная раскладка** - система анализирует структуру графа и автоматически подбирает оптимальный алгоритм визуализации
- **Поддержка множества форматов** - загрузка графов из JSON, MTX, CSV, GEXF, DOT
- **Мета-раскладка** - оригинальный алгоритм иерархической раскладки, учитывающий структуру сообществ в графе
- **Локальный фокус** - поиск важного контекста вокруг выбранного узла с помощью модифицированного алгоритма Дейкстры
- **Интерактивная визуализация** - зум, панирование, фильтрация, подсветка соседей
- **Анализ метрик** - вычисление дегри централов, центральности собственного вектора, модульности и других характеристик
- **Поиск сообществ** - автоматическое разбиение графа на кластеры с помощью алгоритма Leiden
- **WebGL рендеринг** - отрисовка на видеокарте для высокой производительности с большими графами

## Технологический стек

- **Frontend**: TypeScript, Vite
- **Визуализация**: [Sigma.js](https://www.sigmajs.org/) (WebGL-based)
- **Работа с графами**: [Graphology](https://graphology.js.org/)
- **Анализ графов**:
  - Поиск сообществ: Leiden Algorithm ('@aflsolutions/graphology-communities-leiden')
  - Раскладки: Force-directed, Meta, Spectral, Circular, Radial, Hierarchical
  - Метрики: centrality, k-core, eigenvector centrality

## Ключевые идеи

### Мета-раскладка - иерархическая разработка

Традиционные алгоритмы раскладки (Force-directed, Spectral) часто не справляются с большими графами или визуально не отражают их структуру. **Мета-раскладка** решает эту проблему:

1. **Декомпозиция** - граф разбивается на сообщества (плотные подграфы)
2. **Рекурсивная раскладка** - каждое сообщество раскладывается отдельно:
   - Для каждого сообщества вычисляются метрики
   - На основе метрик выбирается подходящий алгоритм
   - Узлы внутри сообщества расставляются оптимально
3. **Мета-граф** - строится граф, где каждый узел - это целое сообщество
4. **Укладка мета-графа** - сообщества раскладываются по отношению друг к другу

**Преимущества:**
- Визуально выделяются сообщества в графе
- Каждое сообщество анализируется с учётом его структуры
- Значительно быстрее (рекурсивная декомпозиция снижает сложность)
- Результаты часто лучше, чем монолитные алгоритмы

### Локальный фокус - контекстный анализ

При клике на узел система находит **важный контекст** вокруг него:

- Используется модифицированный алгоритм Дейкстры
- Находятся узлы, которые функционально связаны с выбранным (не обязательно прямые соседи)
- Система фильтрует граф, показывая только релевантные узлы и рёбра и делая полупрозрачными остальные
- Помогает выявить скрытые связи и выяснить роль узла в глобальной структуре (например, как он связан с сетевыми хабами)

## Требования

- **Node.js** ≥ 18
- **npm** ≥ 10 или **yarn**
- **Docker** & **Docker Compose** (для контейнеризации)

## Быстрый старт

### Без Docker (локальная разработка)

```bash
# 1. Клонировать репозиторий
git clone <repo-url>
cd graph-visualizer

# 2. Установить зависимости
npm install

# 3. Запустить dev-сервер
npm run dev

# Приложение откроется на http://localhost:5173
```

### С Docker Compose

```bash
# 1. Убедитесь, что установлены Docker и Docker Compose
docker --version
docker-compose --version

# 2. Запустить приложение
docker-compose up

# 3. Откройте браузер на http://localhost:5173
```

### Сборка для production

```bash
# Локально
npm run build
npm run preview

# Через Docker
docker build -t graph-visualizer .
docker run -p 5173:5173 graph-visualizer
```
